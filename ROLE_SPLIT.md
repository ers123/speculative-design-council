# 워크숍 역할 분담 — HarmonyOn vs 박은선 교수님

> 60분 워크숍 진행 시 두 분의 역할을 시간대별로 정리.
> 워크숍 당일 일찍 만나 같이 한 번 점검.
>
> **핵심 전제**: Streamlit Council 설치는 학생 선택사항. **HarmonyOn 노트북 1대로 시연**, 학생 대다수는 폰 + 브라우저 + 종이로 따라옴. 진행 흐름은 *"강의실 앞 시연 → 학생 짝꿍 활동"* 의 반복.

---

## 핵심 분담

| 역할 | HarmonyOn (외부 facilitator) | 박은선 교수님 (강사) |
|---|---|---|
| **메인 책임** | 도구 시연 (본인 노트북 1대) + 페다고지 의도 설명 + 시도해본 학생 1:1 도움 | 워크숍 메인 진행 + 학생 narratype·작품 비평 |
| **무대 위치** | 강의실 앞 (시연 시간) → 옆 (학생 짝꿍 활동 시간) | 강의실 앞 (메인 진행), 학생 사이 돌며 코칭 |
| **발언량** | Hook 5분 + 도구 시연 약 15분 + 마무리 5분 (총 ~25분) | 중간 35분 (학생 활동 진행, 비평) |
| **학생과의 관계** | 도구 만든 외부 전문가 (1회성) | 학기 내내 봐 온 강사 |

---

## 시간대별 진행

### 0–5분 · Hook (HarmonyOn 주도)
**HarmonyOn**:
- 자기 소개 (1줄): *"AI 도구 만든 사람"*
- 도구 한 줄: *"4-에이전트 Council, 모두 로컬, 데이터 외부 안 나감"*
- 메시지 한 줄: *"오늘 메시지는 AI가 도구가 아니라 디자인 재료라는 것"*
- 거울 vs 지도 한 마디: *"폰의 작은 AI(거울) 와 브라우저 큰 AI(지도) 사이를 본인이 디자인하라"*
- *"오늘은 제가 노트북으로 도구를 보여드리고, 여러분은 폰·브라우저·종이로 따라오시면 됩니다"* — 학생 부담 즉시 낮춤
- *"교수님이 본격 진행 시작합니다"* 인계

**교수님**:
- HarmonyOn 옆에서 청취 + 학생들 환영

---

### 5–15분 · 데모 시연 (HarmonyOn 노트북, 프로젝터)
**HarmonyOn**:
- 강의실 앞 프로젝터에 노트북 화면 미러링
- Streamlit Council 한 라운드 시연 (데모 시나리오 4종 중 1개) — 약 10분
  - Stage 1 Mirror/Map/Chairman 출력 보여줌
  - Stage 2 AGENTS.md draft 생성 시연
  - Persona Council 4명 추천 + 1인칭 반응 한 명만 보여줌
- *"이 도구 설치한 학생은 직접 돌려보세요, 안 한 학생은 폰·브라우저로 같은 흐름 따라하면 됩니다"*

**교수님**:
- 옆에서 청취, 학생 반응 확인
- 수업 후 본인 활용 위해 시연 흐름 메모

**(설치 시도한 학생):** 본인 노트북에서 따라 돌려봄 — 막히면 손 들기

---

### 15–50분 · 메인 활동 (교수님 주도)
**교수님**:
- 학생들에게 다음 흐름 안내:
  1. (15–35분) Telephone Round — **폰 Edge Gallery ↔ 브라우저 ChatGPT/Claude** 수동 copy-paste 5라운드. *"무엇이 사라지고 무엇이 나타났는가"* 메모.
  2. (35–50분) AGENTS.md 작성 — 종이 worksheet 또는 (설치한 학생은) Streamlit Council Stage 2
- 학생 사이를 돌며 narratype 깊이·다양성·비판적 측면 코칭
- 학생 작품에 대한 비평 코멘트 (강사의 전문성 활용)
- 4 도메인 예시 (HAPPY GATE / COCOO / SPLIT / SelectON) 중 본인 분야 가까운 것 참고 권유

**HarmonyOn**:
- 강의실 뒤·옆에서 대기
- 학생이 *"도구 어떻게 쓰는지"* / *"설치한 게 안 돌아가요"* 물어보면 1:1 응답
- *"내용 깊이"* 같은 페다고지 질문은 교수님께 패스
- 본인 노트북 Streamlit Council 켜둔 채 — 학생이 본인 narratype 데모용으로 한 번 돌려보고 싶다 하면 사용
- BACKUP_PLAN 시나리오 발생 시 즉시 개입

---

### 50–60분 · Constellation + 마무리 (교수님·HarmonyOn 공동)
**교수님**:
- 학생 3-4명 짧게 발표 — 본인 narratype + AGENTS.md 핵심 1줄
- 학생들이 STEEP × Dator 4사분면 (성장/붕괴/지속/변혁) 위에 본인 점 plot
- 클래스 단위 별자리 보이기 → 패턴 발견 ("이 학기 학생들이 가장 자주 본 미래 타입은?")

**HarmonyOn (마무리 5분)**:
- *"이건 시작이고, 자기 작품에 발전시켜 가세요"* 메시지
- 도구 GitHub URL 다시 안내, 학기 중 자율 활용 권장
- 학기말 Class Constellation 시각화 (모든 학생 작품 통합) 약속
- 감사 인사

## 시연 도중 발생 가능한 미세 상황

| 상황 | HarmonyOn 응답 | 교수님 응답 |
|---|---|---|
| 학생: *"왜 4명 AI죠?"* | *"WHY.md 보세요 — 디자인 의도 적어둠"* (또는 30초 요약) | 학생 narratype 진도 봐줌 |
| 학생: *"AGENTS.md 잘 모르겠어요"* | *"AGENTS_MD_GUIDE.md + HAPPY GATE 예시 보세요"* | 학생 본인 narratype에 어떻게 적용할지 brainstorm |
| 학생: *"narratype 200자 못 썼는데..."* | NARRATYPE_WORKSHEET 인쇄본 건네줌 | 1:1 1-2분 brainstorm |
| 학생: *"설치한 게 안 돌아가요"* | 본인 노트북 보여주거나, 워크숍 끝나고 5분 도움 약속 | 진행 흐름 유지 |
| 학생: *"저는 설치 안 했는데 괜찮아요?"* | *"네, 제 노트북 시연 보고 따라하시면 됩니다. 폰·브라우저로 충분합니다"* | 진행 흐름 유지 |
| 학생: *"이거 학기 끝나고도 쓸 수 있나요?"* | GitHub repo 안내, 본인이 직접 사용 가능 — 학기 중 INSTALL.md 따라하면 됨 | 학기 중 follow-up 약속 가능 |
